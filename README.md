
# TP Ingeniería de Software 1

Para comitear usamos este formato:

## Tipos

| Tipo | Cuándo usarlo |
|------|---------------|
| `feat` | Nueva funcionalidad |
| `fix` | Corrección de un bug |
| `refactor` | Cambio de código sin modificar comportamiento |
| `docs` | Cambios en documentación |
| `test` | Agregar o modificar tests |
| `chore` | Mantenimiento: dependencias, configuración, etc. |
| `style` | Formato, espacios, comas (sin cambiar lógica) |

---

## Ejemplos

```
feat(auth): agregar login con Google
fix(pipe): mover creación del pipe dentro del loop
refactor(main): separar lógica de fork en función aparte
docs(readme): agregar instrucciones de compilación
chore: actualizar dependencias
style: corregir indentación en main.c
```
